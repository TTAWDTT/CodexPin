# CodexPin 核心功能测试清单（手动 + 脚本）

## 1. 会话计时正确性

- [ ] 在代码中调用 `startSession`，观察计时从 0 开始递增，单位为秒。
- [ ] 一段时间后调用 `stopSession`，确认计时停止且不会继续累加。
- [ ] 会话停止后再次调用 `startSession`，确认计时重新从 0 开始。

## 2. 周度预算累积与重置

- [ ] 通过测试脚本模拟多次会话（`tests/state.test.js`），确认 `weeklyUsedMinutes` 正常累积。
- [ ] 手动检查 `renderer` 中剩余额度文案是否与预期的计算结果一致。
- [ ] 人为修改持久化 JSON 中的 `weekStartDate` 为过去周，再次启动时确认周度统计被重置。

## 3. 状态行队列行为

- [ ] 通过状态接口连续追加 5 条以上的状态文本，确认界面上始终只显示最近 4 条。
- [ ] 确认最旧的状态文本被正确丢弃，顺序与追加顺序一致。

## 4. 持久化恢复

- [ ] 启动应用，执行若干会话操作，使周度用时产生变化。
- [ ] 关闭应用，再次启动，确认周度用时和剩余额度与上次退出前一致。
- [ ] 删除状态 JSON 文件后启动，确认应用退回到干净的初始状态而不会崩溃。

## 5. 窗口置顶与拖动（需在 Electron 环境下手动验证）

- [ ] 启动应用后，将其他窗口切到前台，确认 CodexPin 小部件仍然保持在最上层显示。
- [ ] 使用鼠标拖拽顶栏，确认窗口可以流畅移动，无明显卡顿或闪烁。

## 6. 模式切换（完整 / 极简）

- [ ] 双击顶栏，确认状态区域隐藏，仅保留顶栏（极简模式）。
- [ ] 再次双击顶栏，确认状态区域重新出现（完整模式）。
- [ ] 重启应用后模式保持与上次退出时一致。

## 7. 低额度提示与待命状态

- [ ] 在测试环境中将周剩余时间设置为低于 60 分钟，确认顶栏剩余额度文字样式发生变化（颜色变为强调色）。
- [ ] 当 session 处于待命状态时，左侧时间展示为“待命”或 0，不再继续累加时间。
- [ ] 从待命切回激活状态后，计时重新开始，显示正常。

## 8. 整体稳定性

- [ ] 连续运行应用 30 分钟以上，反复切换会话状态与模式，确认未出现未捕获异常或 UI 死锁。
- [ ] 在状态 JSON 文件为空或部分损坏的情况下启动应用，确认应用能够自动恢复并安全运行。
